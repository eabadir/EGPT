5. Conditional Additivity Formula

The Bernoulli i.i.d. process yields a joint distribution on 2^n strings. Let Sₙ be all bit-strings of length n, P(s)=p^{#1}(1-p)^{#0}. Then:

H(X₁,…,Xₙ) = - Σ_{s∈Sₙ} P(s) log₂ P(s)

Independence ⇒ H(X₁,…,Xₙ₊₁) = H(X₁,…,Xₙ) + H(Bernoulli(p)). Recursively: Hₙ = n · H₁.

Show Algebraic Expansion 
Hₙ₊₁ = - Σ_{s∈Sₙ} Σ_{b∈{0,1}} P(s) P(b) log₂ (P(s) P(b))
			= - Σ_{s} Σ_{b} P(s) P(b) [log₂ P(s) + log₂ P(b)]
			= - Σ_{s} P(s) log₂ P(s) Σ_{b} P(b) - Σ_{b} P(b) log₂ P(b) Σ_{s} P(s)
			= Hₙ + H₁.

6. Unique Entropy ⇔ Unique Shannon Code

Rota's argument (and the Lean formalization) shows that once an entropy function satisfies the axioms, it is forced to be (a constant multiple of) Shannon entropy. For an i.i.d. Bernoulli process this means: the set of outcome probabilities at depth n already determines the distribution's entropy, and therefore determines the ideal real-valued code lengths ℓ(s) = -log₂ P(s).

Each new trial refines every prior code word into two children (branch 0 and branch 1). No new probability mass can be formed as a sum of a previous subset while also preserving the recursive uniformity condition on every partition (each node splits into probabilities proportional to (1-p):p). Thus every added trial creates a new set of code lengths uniquely determined by the new factor of either -log₂ p or -log₂(1-p). In the special uniform case p = 1/2, every outcome at depth n has probability 2^{-n} and code length exactly n; the tree depth itself gives the unique code length and equals log₂(2^n).

7. Incremental Shannon Encoder Efficiency

Section 6 established that each additional trial appends an independent factor of either -log₂ p or -log₂(1-p) to every existing code. An ideal Shannon encoder therefore only needs the previous code length and the new event to update: ℓₖ₊₁(s·b) = ℓₖ(s) + (-log₂ P(b)). 8. Bijection: Shannon Encoding ⇔ List Bool ⇔ Numbers

Lean formalizes a constructive bijection between finite binary sequences (List Bool) and natural numbers (Section: EGPT number theory). Each binary list is both: (a) a compressed Shannon code word for an equiprobable set of outcomes of size a power of two, and (b) the canonical binary representation of a natural number.

Thus storage efficiency = computational efficiency = arithmetic. Appending one Bernoulli trial bit simultaneously: (1) doubles the addressable outcome space, (2) increases entropy by exactly 1 bit in the uniform case, and (3) increments the maximal representable natural number from 2^k - 1 to 2^{k+1} - 1. The logarithm isn't just measuring; it is the encoder length function forced by the axioms.

9. Informationally Irreducible Shannon Codes

At each depth n the Bernoulli process produces a multiset of probabilities { p^k (1-p)^{n-k} | k=0..n } with multiplicities \binom{n}{k}. These are the informationally irreducible code classes introduced at step n: no new probability value can be obtained as a sum of distinct probability values from the previous step n-1 while still preserving the required recursive partition uniformity (factorization by p or 1-p at each node). 10. Prime Information Atoms (EGPT–FTA Interactive)

This section illustrates the EGPT reinterpretation of the Fundamental Theorem of Arithmetic: multiplicative prime factorization becomes additive decomposition of information. With the calibration that a fair coin has one bit of information, the entropy / information content of a uniform system of k states is log₂ k. Thus for any natural number n ≥ 2 we have:

H(n) = log₂ n and if n = ∏ pᵢ ^ eᵢ (prime factorization) then log₂ n = Σ eᵢ · log₂ pᵢ. For factorials: log₂(n!) = Σ_{p ≤ n} ( Σ_{k≥1} ⌊ n / p^k ⌋ ) log₂ p.
