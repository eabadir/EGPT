4. Conditional Additivity Formula

The Bernoulli i.i.d. process yields a joint distribution on 2^n strings. Let Sₙ be all bit-strings of length n, P(s)=p^{#1}(1-p)^{#0}. Then:

H(X₁,…,Xₙ) = - Σ_{s∈Sₙ} P(s) log₂ P(s)

Independence ⇒ H(X₁,…,Xₙ₊₁) = H(X₁,…,Xₙ) + H(Bernoulli(p)). Recursively: Hₙ = n · H₁.

Show Algebraic Expansion 
Hₙ₊₁ = - Σ_{s∈Sₙ} Σ_{b∈{0,1}} P(s) P(b) log₂ (P(s) P(b))
			= - Σ_{s} Σ_{b} P(s) P(b) [log₂ P(s) + log₂ P(b)]
			= - Σ_{s} P(s) log₂ P(s) Σ_{b} P(b) - Σ_{b} P(b) log₂ P(b) Σ_{s} P(s)
			= Hₙ + H₁.

6. Unique Entropy ⇔ Unique Shannon Code

Rota's argument (and the Lean formalization) shows that once an entropy function satisfies the axioms, it is forced to be (a constant multiple of) Shannon entropy. For an i.i.d. Bernoulli process this means: the set of outcome probabilities at depth n already determines the distribution's entropy, and therefore determines the ideal real-valued code lengths ℓ(s) = -log₂ P(s).

Each new trial refines every prior code word into two children (branch 0 and branch 1). No new probability mass can be formed as a sum of a previous subset while also preserving the recursive uniformity condition on every partition (each node splits into probabilities proportional to (1-p):p). Thus every added trial creates a new set of code lengths uniquely determined by the new factor of either -log₂ p or -log₂(1-p). In the special uniform case p = 1/2, every outcome at depth n has probability 2^{-n} and code length exactly n; the tree depth itself gives the unique code length and equals log₂(2^n).

8. Bijection: Shannon Encoding ⇔ List Bool ⇔ Numbers

Lean formalizes a constructive bijection between finite binary sequences (List Bool) and natural numbers (Section: EGPT number theory). Each binary list is both: (a) a compressed Shannon code word for an equiprobable set of outcomes of size a power of two, and (b) the canonical binary representation of a natural number.

Thus storage efficiency = computational efficiency = arithmetic. Appending one Bernoulli trial bit simultaneously: (1) doubles the addressable outcome space, (2) increases entropy by exactly 1 bit in the uniform case, and (3) increments the maximal representable natural number from 2^k - 1 to 2^{k+1} - 1. The logarithm isn't just measuring; it is the encoder length function forced by the axioms.

9. Informationally Irreducible Shannon Codes

At each depth n the Bernoulli process produces a multiset of probabilities { p^k (1-p)^{n-k} | k=0..n } with multiplicities \binom{n}{k}. These are the informationally irreducible code classes introduced at step n: no new probability value can be obtained as a sum of distinct probability values from the previous step n-1 while still preserving the required recursive partition uniformity (factorization by p or 1-p at each node). 12. Rota's Uniqueness of Entropy & Factoring With Shannon Codes: Entropy IS THE NUMBER OF HEADS in the Bernoulli Process. The Shannon Code RECORDS THE NUMBER OF HEADS

This section links Sections 9–11: interpret the chosen integer N as a bias-coded probability. Let k = bitlength(N) (so 2^{k-1} ≤ N < 2^{k}) and define the bias b_N = N / 2^{k}. Since the leading bit of N is 1, we always have 1/2 ≤ b_N < 1. We treat the (toy) "entropy" of a code here as its popcount (number of 1 bits) to emphasize a discrete counting analogue: more 1's ⇒ larger weight towards the upper half of the unit interval.

Goal (a): Find factor pairs P·Q = N where bitlength(P)+bitlength(Q)=k so that b_P · b_Q = b_N with b_X = X / 2^{bitlength(X)}. These are bias-preserving factorizations: multiplicative structure aligns with additive bit-length budget exactly (no overflow bit). Most factorizations have bitlength(P)+bitlength(Q) > k; only special pairs saturate the budget.

How to read Section 12

1. We reinterpret N as sitting in the interval [0,1) by scaling with its own power-of-two boundary. The fractional part b_N reflects how "biased toward 1" the binary word is, normalized by length.
2. A bias-preserving factorization discovers structure: two shorter words whose normalized magnitudes multiply exactly to the original bias without needing an extra bit. This is rarer for random N.
3. The code class table shows that setting a Bernoulli coin to probability b_N recreates the same average bias across its n-length strings; concentration (variance row) explains stability as n grows.
4. Pedagogical moral: multiplicative arithmetic structure (factors) and additive probabilistic structure (bit-length, expected #1s) meet precisely at bias-preserving factorizations.
